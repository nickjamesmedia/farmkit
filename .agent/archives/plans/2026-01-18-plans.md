# Plans

## Active plan: Wire v0.1 schema to current frontend pages
**Status:** Completed  
**Owner:** Codex  
**Started:** 2026-01-12

### Goal
Connect the existing frontend pages to the v0.1 database schema so core flows load and persist data using the intended tables and relationships.

### Non-goals
- Redesigning page layouts or UX flows.
- Changing the v0.1 schema or RLS policies (unless explicitly required and approved).
- Introducing new modules beyond Equipment, Buildings/Locations, and Maintenance Logs.

### Assumptions
- The v0.1 schema is already defined and is the source of truth for data access.
- Current frontend pages exist but may still point at prototype data structures or placeholders.
- Supabase remains the backend for v0.x.

### Work items (ordered)
1) Inventory current frontend pages and identify data dependencies and any prototype-era data shapes.
2) Map each page to the v0.1 schema tables/relationships and note required query changes.
3) Update data access wiring to read/write against the v0.1 schema (one page/flow at a time).
4) Validate core flows for Equipment, Buildings/Locations, and Maintenance Logs using the updated schema.
5) Update `.agent/TASKS.md` with progress and any follow-up work.

### Acceptance criteria
- Each core page loads data from the v0.1 schema without placeholder/mock data.
- Create/edit flows for core modules persist to the v0.1 schema.
- No schema changes made without an explicit decision and plan update.

### Validation
- Manual smoke checks of core pages (Equipment, Buildings/Locations, Maintenance Logs) against a test dataset.
- If automated checks exist, run the minimal available frontend test/build commands and record results.
